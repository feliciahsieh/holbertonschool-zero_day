  <h2>Readme</h2>

<p>Read the page <a href="/rltoken/TvhnXN1GAP7Et5OSuceGqw" title="Basic vi Commands" target="_blank">Basic vi Commands</a>.</p>

<h2>What you should learn from this project</h2>

<p>At the end of this project you are expected to be able to explain to anyone, <strong>without the help of Google</strong>:</p>

<ul>
<li>What is vi</li>
<li>Who is Bill Joy</li>
<li><a href="/rltoken/TvhnXN1GAP7Et5OSuceGqw" title="The basic vi commands" target="_blank">The basic vi commands</a>

<ul>
<li>How to start and exit vi</li>
<li>What are the command and insert modes, and how to switch from one to the other</li>
<li>Edit text</li>
<li>Cutting and pasting lines</li>
<li>Searching forward and backward</li>
<li>Undo</li>
<li>Quitting</li>
</ul></li>
</ul>

<h2>Requirements</h2>

<ul>
<li>Allowed editors: vi</li>
<li>A <code>README.md</code> file, at the root of the folder of the project, describing what this project is about</li>
<li>If the answer of one of the following questions needs the <code>Return</code> key, then use the string <code>&lt;Return&gt;</code>. Example: What is the command to write current contents to file named in original vi call? Answer should be exactly <code>:w&lt;Return&gt;</code>. Note that there is no space between <code>w</code> and <code>&lt;Return&gt;</code></li>
<li>Your file should only contain the command, and nothing else</li>
</ul>
